관계 연결 및 편집

class 다이어그램은 class의 palett에서 객체를 이용하여 작성한다.

  1. 연관 관계

    2. 연관 관계 라인은 연관의 종류에 따라 단순 연관, 집합, 조합으로 세분화된다. 연관의 종류는 팝업 메뉴를 통해 변경할 수 있다. 연관의 종류가 변경되면 라인의 화살표 모양이 변경되어 연관의 종류를 시각적으로 표현한다.


    그림 2.1-2 연관 관계 종류

  2. 의존 관계

    3. 한 클래스의 기능 수행 가능 여부가 다른 클래스에 전적으로 의존하고 있는지를 표현한다.


    그림 2.1-2 의존 관계

  3. 상속 관계

    4. 두 클래스가 일집화-특수화 관계가 있는지를 표현한다. source 클래스가 target 클래스로 상속 특성을 가진다.


    그림 2.1-2 상속 관계

  4. 관계 편집

    5. 관계에 마우스를 갖다 대고 우클릭을 하면 다음과 같은 팝업창이 뜬다.


    그림 2.1-2 관계 팝업창

    • Aggregate - 집합 연관관계를 표현한다.
    • Composite - 복합 연관관계를 표현한다.
    • Vavigable - 방향성 관계를 표현한다.
    • Role Name - 관계에 Role Name을 설정한다.
    • Direct Line - 관계를 직선으로 만든다.
    • Delete - 관계를 삭제한다.

    관계의 프로퍼티창은 이름, 프로퍼티, 스테레오타입으로 나뉘는데 프로퍼티에서는 세부 내용을 설정할 수 있다. SourceRole과 TargetRole는 대상이 다르고 같은 내용이며 Description은 관계에 대한 내용을 변경할 수 있다.


    그림 2.1-2 관계 프로퍼티

    • Role - role 이름을 변경한다.
    • RoleNote - role에 설명내용을 변경한다.
    • Constraint(s) - 추가 작업 필요
    • Qualifier(s) - 추가 작업 필요
    • Derived - 추가 작업 필요
    • Derived Union - union을 설정한다.
    • Owned - 추가 작업 필요
    • Ordered - 추가 작업 필요
    • Allow Duplicates - 추가 작업 필요
    • Multiplicity - 추가 작업 필요
    • Aggregation - 관계 타입을 설정한다.
    • Navigability - 관계 방향성을 설정한다.
